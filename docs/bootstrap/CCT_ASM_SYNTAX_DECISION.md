# CCT ASM Syntax Decision — FASE 16C.3

- Status: Normative (active)
- Scope: CCT `--emit-asm` pipeline in freestanding profile
- Source of truth: `md_out/FASE_16_CCT.md` (16C.3), `docs/bootstrap/CCT_LOWERING_MATRIX_V0.md`

## Official Decision

The official assembly format generated by CCT in FASE 16 is:

- Intel syntax for GAS, with mandatory `.intel_syntax noprefix` header;
- official assembly through `as --32` (binutils).

The `.intel_syntax noprefix` line must appear at the top of every `.cgen.s` file.

## Technical Rationale

1. The current pipeline uses cross-GCC with `-m32 -S -masm=intel`, which generates GAS-compatible assembly.
2. The flow reduces regression risk by avoiding additional dialect conversion in this phase.
3. Integration with automatic validation (16C.2) already guarantees ABI/lowering invariants over this format.

## Environment Requirements

- `as` (GNU assembler) with `--32` support.
- `nm` and `objdump` for artifact auditing.
- Minimum recommended binutils: compatible with `.intel_syntax noprefix` (common in modern versions; historical reference >= 2.18).

## Canonical Command

```bash
as --32 <file>.cgen.s -o <file>.o
```

## Non-Goals for FASE 16

- Do not implement a GAS -> NASM converter in this phase.
- Do not change the backend to native NASM emission.

NASM conversion is explicitly deferred to a future phase if direct integration with a pure NASM toolchain becomes necessary.

## Implications for LBOS

- The CCT-LBOS bridge in FASE 16 uses `.cgen.s` in Intel-syntax GAS and assembly via `as --32`.
- Symbol and ABI contracts remain governed by:
  - `docs/bootstrap/CCT_SYMBOL_NAMING_V0.md`
  - `docs/bootstrap/CCT_ABI_V0_LBOS.md`
  - `docs/bootstrap/CCT_LOWERING_MATRIX_V0.md`

## Expected Test Evidence

1. `.cgen.s` contains `.intel_syntax noprefix` at the top.
2. `as --32` assembles `.cgen.s` with exit code 0.
