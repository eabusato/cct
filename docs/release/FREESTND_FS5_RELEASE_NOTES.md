# Freestanding FS-5 Release Notes

## Summary

FS-5 closes the first bare-metal HTTP-server block of the CCT freestanding track.

Gate closed:

- `G-FS5`: the booted system now serves HTTP on bare metal and returns HTML generated by CCT for `GET /`

## Highlights

### FS-5A

- `../grub-hello/src/http_server.c` added the poll-based HTTP accept/read/close lifecycle on top of the existing TCP layer
- `cct/http_server_fs` became the canonical CCT surface for accept loops, request buffering, and request counters

### FS-5B

- `../grub-hello/src/http_parser.c` added request-line, header, query, and body parsing without host heap services
- `cct/http_parser_fs` was delivered as the canonical freestanding request-inspection module

### FS-5C

- `../grub-hello/src/http_response.c` added the freestanding HTTP response builder with explicit `Content-Length` and `Connection: close`
- `cct/http_response_fs` was delivered with direct helpers plus staged header/body assembly

### FS-5D

- `../grub-hello/src/http_router.c` added exact/prefix routing with handler ids resolved in CCT
- `../grub-hello/src/civitas_boot_fase5.cct` now boots into the FS-5 HTTP screen and serves `/`, `/status`, `/api/info`, `/echo`, and a default 404 page
- the final gate is validated by a raw Ethernet/TCP/HTTP probe against the booted RTL8139 guest

## Compatibility Notes

- host-profile behavior remains unchanged; `cct/http_server_fs`, `cct/http_parser_fs`, `cct/http_response_fs`, and `cct/http_router_fs` are freestanding-only
- `../grub-hello` still preserves the compatibility build artifacts under `build/civitas_boot.*`
- the active freestanding entry in the integration repo is now `civitas_boot_fase5`

## Verification

- filtered block: `CCT_TEST_PHASES=FS5A,FS5B,FS5C,FS5D bash tests/run_tests.sh`
- boot integration: `cd ../grub-hello && make iso`
- live gate probe: `python3 tests/support/freestanding_fs5_gate_probe.py --qemu-bin qemu-system-i386 --iso ../grub-hello/grub-hello.iso`
- full suite: `bash tests/run_tests.sh`
